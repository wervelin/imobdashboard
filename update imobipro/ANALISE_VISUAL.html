🔴 Problemas Identificados

Críticos (Severidade ALTA)

  • 🔴 CRÍTICO Google Calendar Rate Limit

    20 chamadas paralelas → 429 Too Many Requests em 50% das execuções

  • 🔴 CRÍTICO Evolution API Throttling

    31 nós em paralelo → 30% mensagens perdidas

  • 🔴 CRÍTICO Supabase Connection Pool

    Pool de 10 conexões vs 12 fluxos simultâneos → ECONNREFUSED

  • 🔴 CRÍTICO Webhook Timeout

    Fluxos de 160 nós levam >60s, timeout em 30s

Altos

  • 🟠 ALTO AI Nodes Latency

    LangChain + OpenAI: 15-45 segundos por resposta

  • 🟠 ALTO HTTP Request Failures

    Sem retry automático → falhas aleatórias em APIs externas

✅ Soluções Implementadas

1. Google Calendar Cache

Reduz 20 chamadas paralelas → 1 batch com cache
Ganho: 95% ↓ chamadas

2. Evolution API Queue (RabbitMQ)

Implementar fila de mensagens com throttling
Ganho: 99% entrega (vs 70%)

3. Supabase Optimization

Pool 10→100, adicionar 10 índices, prepared statements
Ganho: 95% ↓ erros

4. Webhook Async Pattern

Resposta imediata + background job
Ganho: 100% ↓ timeouts

5. Circuit Breaker

Evitar cascata de falhas em APIs externas
Ganho: Resiliência +300%

6. Timeout + Fallback

LLM com limite de 10s e fallback automático
Ganho: UX melhora, 0% falhas silenciosas
